title: '各种各样的宝可梦叫声从程序创意中诞生'
diff_colors: '增田 #F48624'
---

**——给在《红／绿》登场的 151 种宝可梦制作叫声想必是一项很艰巨的工作，而且 ROM 容量的限制也很苛刻吧？**

增田：实际上，《红／绿》中宝可梦叫声的基础数据大约只有 40 来种，各种各样的叫声都是先有一个最初设想出来的基础声音，然后通过程序改变其速度或者音高制作出来的。例如，妙蛙种子、妙蛙草、妙蛙花这 3 种宝可梦，它们叫声的基础音就是同一个，正是通过程序处理来制作出它们各自的叫声。毕竟这是一条进化链，我想这样的制作思路也很合理对吧，体型变大就把叫声调低一些，或者来个出其不意的反向操作。又比如皮皮的，虽然听起来就真的像“哔哔”那样叫，但其基础音是完全不一样的，你现在听到的“哔哔”其实就是将基础音调成高速、再把音调调高之后的样子。

**——好像过去的怪兽电影里，就是通过改变录影带的播放速度、或者倒带来制作怪兽叫声的，你说的方法和这种很接近啊。**

增田：我、田尻还有杉森，都是看昭和特摄、怪兽电影长大的，我们有一个共识，就是认为“怪兽、怪物的叫声应该就是这样的一种感觉”，所以最先就是以这种容易理解的“怪兽画风”作为起点来制作叫声。但问题来了，像毛毛虫那样的宝可梦又应该是怎么叫的呢？于是我们又果断将之归类为“发出特殊叫声的宝可梦”，然后就都按照这种理念去制作叫声了。我也一直在考量，叫声如何变换处理才能让人听起来最合适和有趣。

**——就是说，为了使叫声能够更丰富多样，是在程序方面下的创意工夫吗？**

增田：没错，宝可梦的叫声使用了两个通道的方波外加噪声，而 Game Boy 的方波又有 4 种不同的波形。于是乎每 1/30 秒切换一次波形，这样来增强叫声的表现力，如此工夫可下了不少。要是没有这个系统，我想也就做不出来这种叫声了。另外，叫声播放期间，背景的低音也是在持续播放的（一共 4 通道，叫声用了 3 个，低音位于剩下的一个），以免让人觉得音乐中断。
